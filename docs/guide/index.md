# Guide

`phenotype-xdd` is a documentation-only hub. It exists to organize methodology references, architecture decisions, and engineering practices.

## What Lives Here

- Methodology summaries and cross-references
- Architecture decision records
- Functional requirements tied to numbered practices
- Locale entry points for a predictable public surface

## How To Contribute

1. Update the canonical Markdown source in this repo.
2. Keep practice IDs stable and unique.
3. Add or update site navigation when a new doc surface is introduced.
4. Validate the docs build before landing changes.

## Navigation

- [Reference](/reference/) for methodology summaries
- [PRD](/reference/prd) for goals and epics
- [ADR](/reference/adr) for hub decisions
